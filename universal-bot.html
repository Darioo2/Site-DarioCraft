Guide d'utilisation

Tout ce qu'il faut savoir pour utiliser Universal Bot.

Par défaut, un projet vide avec une macro "Macro 1" est créé au lancement.

Créer et enregistrer une macro

Une macro est une séquence d'actions (clics, touches) que vous pouvez rejouer à volonté.

Enregistrer vos actions

  1. Cliquez sur ⏺ ENREGISTRER
  2. Effectuez vos actions sur l'écran (clics, frappes clavier…)
  3. Appuyez sur Échap ou cliquez STOP pour terminer
  4. Vos actions apparaissent dans la liste centrale
Si des actions existaient déjà, l'application vous demande :
Oui = insérer les nouvelles actions après la sélection actuelle
Non = tout effacer et recommencer

Éditer une action manuellement

  • Double-clic sur une action → ouvre l'éditeur de l'action
  • Clic droit sur une action → menu : Modifier, Dupliquer, Déplacer, Supprimer
  • Menu Édition en haut → Annuler, Rétablir, et autres opérations

Gérer vos macros

  • + Nouvelle macro → ajoute une macro vide
  • Double-clic sur le nom d'une macro → la renommer
  • ✕ à droite du nom → supprimer la macro (impossible si c'est la seule)

Mode Lire

Le bouton ▶ LIRE lit toutes les macros du projet dans l'ordre.

Sans "Répéter en boucle"

Lit Macro 1, puis Macro 2, puis Macro 3… et s'arrête à la fin.

Avec "Répéter en boucle"

Lit Macro 1, 2, 3… puis recommence depuis le début, indéfiniment, jusqu'au clic sur STOP LECTURE.

Décompte avant lecture

Un décompte de 3 secondes s'affiche (Lecture dans 3… 2… 1…) vous laissant le temps de cliquer dans la fenêtre cible. Cliquer à nouveau sur le bouton pendant ce décompte annule la lecture.

Cliquez ⏹ STOP LECTURE à tout moment pour arrêter proprement à la fin de l'action en cours.

Mode Surveiller

Le mode Surveiller parcourt toutes vos macros en boucle et les déclenche automatiquement selon les conditions configurées.

Ce qui se passe

Le moteur examine chaque macro dans l'ordre
↓
Macro inactive → ignorée
↓
Conditions remplies → macro lue en entier
↓
Dernière macro examinée → recommence depuis la première
LIRE et SURVEILLER ne peuvent pas tourner en même temps.

Les déclencheurs

Les déclencheurs définissent quand une macro doit se déclencher pendant la surveillance. Accédez-y via le bouton ⚡ Déclencheurs dans le panneau droit.

Déclencheur Pixel

Surveille la couleur d'un pixel précis de l'écran. Configurez X, Y et la couleur de référence via "Capturer le pixel".

  • Même couleur → se déclenche quand le pixel a exactement cette couleur
  • Couleur différente → se déclenche quand le pixel n'a PAS cette couleur

Revérifié ~100 ms. Condition dynamique : peut redevenir vraie ou fausse à tout moment.

Déclencheur Horloge

Déclenche à une heure précise (format HH:MM:SS, horloge système).

  • Déclencher → à partir de l'heure, condition vraie pour toute la session
  • Arrêter → avant l'heure vraie, à partir de l'heure fausse définitivement

Si la surveillance démarre après l'heure configurée, la condition est immédiatement active.

Déclencheur Timer

Compte à rebours qui démarre au moment où la surveillance commence.

  • Déclencher → quand le compte expire, condition vraie pour toute la session
  • Arrêter → pendant le compte vraie, à l'expiration fausse définitivement

Un nouveau STOP puis SURVEILLER repart le timer de zéro.

Combiner plusieurs déclencheurs

OU

La macro se déclenche si au moins un déclencheur est vrai.

ET

La macro se déclenche seulement si tous les déclencheurs sont vrais en même temps.

Options des macros

Ces options sont accessibles dans la fenêtre ⚡ Déclencheurs de chaque macro.

Macro active
Si décochée, la macro est complètement ignorée pendant la surveillance, même si ses déclencheurs seraient vérifiés.
Répéter en boucle
Après lecture, les déclencheurs sont réévalués. Si la condition est encore vraie → repart. Si elle est devenue fausse → le moteur passe à la macro suivante.
Démarrer au lancement
La macro démarre immédiatement quand la surveillance commence, sans attendre un déclencheur. Combinez avec "Répéter en boucle" et un déclencheur "Arrêter" pour créer un comportement : "tourne dès le départ, s'arrête quand [timer / horloge] se déclenche".
Condition ET / OU
Visible uniquement quand la macro a au moins deux déclencheurs. Voir la section Déclencheurs ci-dessus.

Sauvegarde des projets

Les projets se sauvegardent dans des fichiers .ubot (format texte JSON).

Fichier › Enregistrer Sauvegarde rapide
Fichier › Enregistrer sous… Choisir l'emplacement
Fichier › Ouvrir… Charger un projet existant
Fichier › Nouveau Repartir de zéro
Un ● devant le nom du projet dans la barre de titre indique des modifications non sauvegardées. L'application vous avertit si vous essayez de quitter sans sauvegarder.

Messages de la console

La bande verte en bas affiche l'état en temps réel. Voici les messages courants :

[Macro 1] Condition fausse → En attente La macro attend que sa condition soit remplie.
[Macro 1] Déclencheurs validés → Lecture démarrée La condition est vraie, la macro commence à jouer.
[Macro 1] Lecture terminée La macro a fini de jouer.
[Macro 1] Déclencheurs invalidés → Lecture stoppée La condition est redevenue fausse, la répétition s'arrête.
[Macro 1] Démarrage immédiat → Lecture démarrée La macro démarre grâce à l'option "Démarrer au lancement".
